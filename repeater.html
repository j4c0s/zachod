Instrukcja dla Repeaterów

Zwięzły poradnik: Szybka i zoptymalizowana konfiguracja stacji bazowych (przemienników) na Zachodzie.
⚡ Aktualizacja oprogramowania:
Zaleca się aktualizację do wersji oprogramowania minimum 1.16. Oficjalne wydania pobierzesz ze strony meshcore.io. Zoptymalizowane wersje z oszczędzaniem energii i preambułą 32 pobierzesz bezpośrednio z IoTThinks GitHub Releases.
💻 Wiersz poleceń (CLI):
Po wysłaniu każdej komendy w CLI zawsze poczekaj na zwrotne potwierdzenie jej wykonania z przemiennika.

1. Zasilanie i parametry fizyczne

  • Zasilanie i Bilans solarny: Zweryfikuj stabilność napięcia przy nadawaniu. W instalacjach solarnych upewnij się, że panel i akumulator gwarantują stabilną pracę w nocy.
  • Szumy tła (Noise Floor): RSSI tła powyżej -110 dBm oznacza lokalne zakłócenia. W pobliżu masztów komórkowych (BTS) wysoce zalecane jest użycie filtra szczelinowego (cavity filter).
  • Nazewnictwo: W nazwie węzła nie stosuje się prefixu "RPT". Wystarczy nazwa określająca unikalną lokalizację (np. Sulechow_Wsch), ułatwiająca celowanie anten.

2. Tożsamość i kontakt

Ustaw swój nick lub znak wywoławczy jako właściciel stacji do diagnostyki:

set owner.info "TWÓJ_NICK_LUB_ZNAK"

3. Rozgłoszenia (Adverty)

Rozgłoszenie Lokalne (Zero-Hop)

Ustaw interwał rozgłoszeń lokalnych w minutach (zalecane 4 godziny czyli 240 minut):

set advert.interval 240

Rozgłoszenie Globalne (Flood)

Ustaw interwał globalnych rozgłoszeń w godzinach (zalecane 23 godziny):

set flood.advert.interval 23

4. Synchronizacja zegara

Do synchronizacji czasu systemowego:

clock sync

W przypadku błędnego zegara, którego nie można zsynchronizować tradycyjnie, użyj komendy restartu zegara:

clkreboot

5. Konfiguracja regionów

Do filtrowania niepotrzebnego ruchu radiowego z odległych województw stosuje się system regionów. Dodanie ich do repeatera nie koliduje z dotychczasowym ruchem bez regionu (unscoped), a odblokowuje poprawne powtarzanie wiadomości regionalnych.

🌍 Zasada doboru regionów:
Dodawaj do konfiguracji wyłącznie regiony odpowiadające Twojemu województwu oraz regiony nadrzędne (jak pl czy zachod). Pełną listę oraz taryfikator znajdziesz na stronie Regiony i Scopes.

Poniżej znajdują się schematyczne przykłady komend CLI służące do wprowadzania i zarządzania regionami:

# Wprowadzenie regionu do pamięci:
region put [nazwa_regionu]
# Zezwolenie na floodowanie pakietów z tego zakresu:
region allowf [nazwa_regionu]
# Powiązanie własnych i unscoped pakietów z domyślnym regionem województwa:
set default.region [nazwa_regionu]
# Zapisanie zmian w pamięci stacji i restart:
region save
reboot

6. Optymalizacja i bezpieczeństwo (CLI)

  • Repeat Mode (Tryb powtarzania): Włącza powtarzanie ruchu w sieci. 
    set repeat.mode on
  • Duty Cycle: Ogranicza czas nadawania do 10% (zgodność z europejskimi normami). 
    set dutycycle 10
  • Loop Detect (Wykrywanie pętli): Chroni przed lawinowym zapętlaniem pakietów. 
    set loop.detect minimal
  • Multiple ACKs: Poprawia stabilność potwierdzeń przy słabym sygnale. 
    set multi.acks 1
  • Hasło Gościa: Pozostawienie pustego hasła konsoli pozwala sąsiadom na bezpieczny podgląd diagnostyczny. 
    set guest.password ""

7. Opóźnienia nadawania (txdelay)

Strojenie parametru txdelay chroni sieć przed kolizjami radiowymi. Im wyżej położony i lepiej słyszalny przemiennik, tym dłużej powinien czekać na retransmisję.

>15 sąsiadów		 Wysokie wieże, dachy wieżowców.
