Instrukcja dla Repeaterów

Parametry techniczne, zalecane limity przeskoku (flood) oraz profile opóźnień (txdelay) dla stacji bazowych na Zachodzie.

1. Kroki kontrolne przed montażem

Przed umieszczeniem przemiennika wysoko na dachu lub maszcie, upewnij się, że stacja działa bez zarzutu:

  • Zasilanie i pobór prądu: Sprawdź obciążenie akumulatorów lub zasilacza buforowego.
  • Status GPS Fix: Jeśli moduł ma GPS, upewnij się, że widzi satelity na otwartym terenie.
  • Poziom szumów (Noise Floor): Zmierz wskaźnik RSSI tła. Ciągły poziom szumów powyżej -110 dBm oznacza lokalne zakłócenia utrudniające odbiór.

2. Nazewnictwo Repeaterów

Dobra praktyka: Ze względu na brak sztywnego krajowego standardu, zalecamy w nazwie przemiennika jasno wskazać miejscowość/dzielnicę oraz typ montażu, np. RPT_Wroclaw_Krzyki lub RPT_Luban_Maszt. Ułatwia to błyskawiczną identyfikację tras sygnału.

3. Rozgłoszenia (Adverty)

Aby zapobiec wysycaniu pasma częstymi, powtarzającymi się ramkami statusu, konfigurujemy rzadkie interwały reklam lokalnych oraz globalnych: 

# Status lokalny tylko do bezpośrednich sąsiadów co 4 godziny (240 minut)
set advert.interval 240

# Status globalny przekazywany floodem po sieci co 24 godziny
set flood.advert.interval 24

4. Profile Opóźnień Nadawania (txdelay)

Zróżnicowanie czasów opóźnienia chroni sieć przed kolizjami radiowymi na tym samym obszarze. Im wyżej położony węzeł, tym dłużej powinien odczekać przed retransmisją, pozwalając na nadanie stacjom położonym niżej:

Profil wysokości Zastosowanie Parametry CLI
WYSOKI / MASZT Wysokie wieże, maszty kratownicowe, dachy wieżowców (daleki zasięg) set txdelay 1.5
set direct.txdelay 1.0
set rxdelay 3
DACH / ROOFTOP Klasyczny montaż anteny na dachu domu jednorodzinnego lub niskiego bloku set txdelay 0.8
set direct.txdelay 0.4
set rxdelay 3
DOM / NISKI Węzeł domowy, antena na parapecie lub wewnątrz pomieszczeń set txdelay 0.3
set direct.txdelay 0.1
set rxdelay 3

5. Limity Przeskoku (Hopy)

W celu zredukowania ogólnego szumu w eterze, ograniczamy ruch nieukierunkowany (unscoped) przy jednoczesnym zezwoleniu na szerokie przeskoki dla dobrze zaadresowanych pakietów regionalnych: 

# Maksymalny limit hopów dla pakietów ze scope regionalnym (np. zachod, pl-ds)
set flood.max 64

# Ograniczenie przesyłu dla ruchu ogólnego bez scope (unscoped)
set flood.max.unscoped 8
💡 Wskazówka dotycząca unscoped: Wartość flood.max.unscoped 8 jest wyjściowym standardem. Obserwuj chat w swojej okolicy i dostosuj ten parametr w przedziale 6–8 w zależności od tego, przez ile przeskoków (hopów) docierają do Ciebie pożądane pakiety.

6. Pozostała konfiguracja optymalizacyjna

Wprowadź poniższe komendy w CLI, aby zoptymalizować pracę radia:

# Włączenie trybu routingu i dwubajtowych path-hashów
set path.hash.mode 1
set multi.acks 1

# Ochrona przed zatykaniem silnym sygnałem (resetowanie gainu SX1262)
set agc.reset.interval 500

# Pozostawienie pustego hasła gościa (ułatwia sąsiedzką diagnostykę)
set guest.password ""

# Zapis i restart urządzenia
write
reboot
