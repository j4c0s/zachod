Instrukcja dla Repeaterów

Krok po kroku: Pełna i zoptymalizowana konfiguracja stacji bazowych (przemienników) w regionie zachodnim.
⚡ Ważna aktualizacja oprogramowania:
Gorąco zachęcamy do aktualizowania repeaterów do wersji oprogramowania minimum 1.16. Oficjalne wydania można pobrać bezpośrednio ze strony meshcore.io, a alternatywne i zoptymalizowane wersje oprogramowania od IoTThinks są dostępne na GitHubie.

Repeatery poniżej wersji 1.14 nie przesyłają wiadomości 2 i 3-bajtowych. Ponadto, wersje poniżej 1.16 wysyłają wiadomości z preambułą 16 zamiast 32, co jest kluczowym elementem w dalszej optymalizacji i obniżaniu zużycia energii baterii.
💻 Wiersz poleceń (Command Line - CLI):
Na tej stronie znajdziesz również parametry i ustawienia, które można wprowadzić wyłącznie poprzez Wiersz poleceń (CLI) po zalogowaniu do repeatera. Pamiętaj: po wysłaniu każdej komendy w CLI poczekaj na potwierdzenie jej wykonania. Komendy bez potwierdzenia zwrotnego mogły nie dotrzeć do repeatera.

1. Zasilanie i pobór prądu

Przed fizycznym montażem przemiennika na dachu lub maszcie należy bezwzględnie przetestować i zweryfikować jego bilans energetyczny:

  • Obciążenie zasilania: Dokładnie sprawdź obciążenie akumulatorów lub zasilacza buforowego pod kątem stabilności napięcia pod obciążeniem nadawczym.
  • Ładowanie solarem: W przypadku instalacji zasilanych panelami fotowoltaicznymi upewnij się, że panel słoneczny oraz kontroler ładowania są odpowiednio dobrane do pojemności akumulatora. Sprawdź, czy proces ładowania solarem efektywnie uzupełnia energię w ciągu dnia, zabezpieczając ciągłość pracy urządzenia w nocy i podczas gorszych warunków pogodowych.
  • Poziom szumów (Noise Floor): Zmierz wskaźnik RSSI tła. Ciągły poziom szumów powyżej -110 dBm oznacza lokalne zakłócenia. W eliminacji zakłóceń z pobliskich masztów komórkowych (BTS) wysoce zalecane jest użycie filtra szczelinowego (cavity filter).

2. Nazewnictwo

Dla ułatwienia orientacji użytkowników w topologii sieci i ułatwienia im wycelowania anten, niezwykle istotne jest prawidłowe nazwanie stacji:

  • Lokalizacja GPS: Dobrze jest ustawić przybliżoną lokalizację GPS, aby repeater pojawiał się na mapach sieci.
  • Nazwa węzła: Ze względu na brak sztywnego krajowego standardu, najważniejsze jest podanie lokalizacji pozwalającej łatwo określić położenie przemiennika. W nazwie nie musi znajdować się sztywne określenie "typ montażu", ani skrót "RPT", o ile nazwa jednoznacznie identyfikuje fizyczną lokalizację stacji bazowej.

3. Informacja o właścicielu

Dla ułatwienia diagnostyki sieciowej i identyfikacji stacji bazowych, ustaw swoją nazwę companion (znak wywoławczy lub pseudonim), pod którą można się z Tobą bezpośrednio skontaktować w sieci. Ułatwi to odnalezienie właściciela w razie potrzeby kontaktu lub awarii: 

set owner.info "TWÓJ_NICK_LUB_ZNAK"

4. Rozgłoszenia (Adverty)

Aby zapobiec niepotrzebnemu wysycaniu pasma częstymi i powtarzającymi się ramkami statusu, konfigurujemy rzadkie interwały advertów. Wizualnie dzielimy je na dwa osobne parametry:

Rozgłoszenie Zero-Hop (Lokalne)

Służy wyłącznie bezpośrednim sąsiadom w zasięgu wzroku. Interwał podajemy w minutach: 240 minut (czyli równe 4 godziny).

set advert.interval 240

Rozgłoszenie Flood (Globalne)

Rozprzestrzenia się po całej sieci jako flood ze statusem zero-hop. Interwał podajemy w godzinach: 23 godziny.

set flood.advert.interval 23

5. Lokalizacja

Dobrze jest ustawić przybliżoną lokalizację GPS w konfiguracji przemiennika. Dzięki temu repeater poprawnie pojawi się na mapach, a inni użytkownicy sieci będą mogli łatwiej wycelować swoje anteny kierunkowe w stronę Twojej stacji bazowej.

6. Zegar i jego synchronizowanie

Prawidłowy czas systemowy na repeaterze jest kluczowy dla właściwej interpretacji logów i poprawności routingu. Do synchronizacji czasu służy dedykowane polecenie: 

clock sync

Wskazówka przy problemach z czasem wstecz: W przypadku, gdy czas systemowy na urządzeniu jest błędny i nie da się go cofnąć tradycyjną komendą, należy najpierw zresetować zegar systemowy poleceniem, a następnie wykonać synchronizację: 

clkreboot

Po restarcie zegara wykonaj ponownie synchronizację czasu:

clock sync

7. Hasło gościa (Guest)

Zaleca się pozostawienie hasła gościa jako puste dla konsoli. Umożliwia to sąsiadom bezpieczną diagnostykę i weryfikację stanu stacji bez konieczności zdradzania haseł administracyjnych: 

set guest.password ""

8. Regiony i zakresy (Scopes)

Mechanizm Regional Scope służy do filtrowania i ograniczania ruchu radiowego w sieci, chroniąc Twój przemiennik przed zapchaniem przez niepotrzebne ramki z odległych województw. Dzięki tym regułom, Twój przemiennik pomoże w przekazywaniu lokalnego i krajowego ruchu sąsiadom, jednocześnie odrzucając zbędny szum.

Ważna uwaga dotycząca kompatybilności: Dodanie regionów (scopes) do konfiguracji repeatera nie powoduje żadnych problemów z przesyłaniem dotychczasowych wiadomości bez ustawionego zakresu (unscoped). Urządzenie nadal będzie poprawnie przekazywać tradycyjny ruch, ale dodatkowo zyska możliwość odbierania i przesyłania dalej wiadomości z przypisanym, konkretnym zakresem regionalnym.

Ustawienie domyślnego regionu (default.region):
Komenda set default.region [scope] (np. set default.region zachod) służy do zdefiniowania domyślnego zakresu terytorialnego. Przypisuje ona wybrany region do wszystkich pakietów generowanych bezpośrednio przez sam repeater (np. jego własne statusy i telemetryczne rozgłoszenia) oraz do pakietów, które dotarły do niego jako "czyste" (bez przypisanego żadnego scope'u). Dzięki temu pakiety te zostaną automatycznie "oznakowane" odpowiednim regionem i będą przekazywane zgodnie z polityką filtracji.

🌍 Zasada dobierania regionów:
Pamiętaj, aby do konfiguracji repeatera dodawać wyłącznie regiony odpowiadające Twojemu własnemu województwu (voivodeship) oraz regionom nadrzędnym (jak krajowy pl czy zbiorczy zachod). Wyjątkiem są sytuacje, gdy przemiennik znajduje się blisko granicy województw lub pełni rolę fizycznego mostu (bridge) przenoszącego ruch pomiędzy sąsiadującymi regionami.

Aby dodać dozwolone zakresy regionalne, ustawić domyślny region oraz zezwolić na ich przekazywanie dalej (floodowanie), wykonaj poniższe polecenia w CLI: 

region put pl
region put zachod
region put pl-ds
region put pl-lb
region put wlkp
region put wroc
region allowf pl
region allowf zachod
region allowf pl-ds
region allowf pl-lb
region allowf wlkp
region allowf wroc
set default.region zachod
region save

Pełną listę oraz taryfikator rekomendowanych kanałów i regionów w Polsce Zachodniej znajdziesz na dedykowanej podstronie: Regiony i Kanały.

9. Włączenie przekazywania (Repeat Mode)

Aby repeater poprawnie przekazywał pakiety i wiadomości od innych użytkowników w sieci, musi mieć bezwzględnie włączoną funkcję **Repeat Mode**: 

set repeat.mode on

10. Dodatkowe ustawienia CLI

Poniżej opisano zaawansowane parametry optymalizacyjne, które można skonfigurować tylko za pomocą CLI. Warto je zawsze sprawdzić lub nawet ustawić dla pewności po każdej aktualizacji oprogramowania (ponieważ zmiana firmware czasem przywraca domyślne parametry urządzenia i trzeba je ustawić ponownie):

  • set dutycycle 10: Ogranicza cykl pracy nadajnika (duty cycle) do 10%, co jest bezpośrednio zgodne z europejskimi normami prawnymi dla pasma LoRa ISM (869.4–869.65 MHz). Wymusza to odpowiednio długi czas ciszy radiowej po każdej retransmisji, dbając o wolne pasmo i legalność działania stacji.
  • set flood.max.advert 8: Ogranicza zasięg przesyłanych advertów sieciowych.
  • set path.hash.mode 2: Wymusza dwubajtowy tryb generowania skrótów ścieżek (path-hash), co znacząco zmniejsza ryzyko kolizji identyfikatorów tras w większych sieciach.
  • set multi.acks 1: Aktywuje wielokrotne potwierdzenia odbioru pakietów, co poprawia niezawodność dostarczania danych przy niestabilnym linku.
  • set agc.reset.interval 500: Określa interwał (w milisekundach) resetowania automatycznej kontroli wzmocnienia (AGC) w układzie SX1262, zapobiegając zatykaniu odbiornika przez silne, bliskie sygnały.
  • set flood.max 64: Maksymalny dopuszczalny limit hopów (przeskoków) dla pakietów z przypisanym i dozwolonym zakresem regionalnym (np. zachod, pl-ds).
  • set flood.max.unscoped 8: Ograniczenie przesyłu dla ogólnego ruchu bez przypisanego zakresu (unscoped). Pakiety te mają mniejszy priorytet rozprzestrzeniania się.
💡 Wskazówka dotycząca unscoped:
Wartość limitu flood.max.unscoped można w pełni dostosować do swoich potrzeb poprzez uważną obserwację przychodzących wiadomości. Im mniejsza wartość, tym mniejsze obciążenie generuje ruch dla całej lokalnej sieci. Najlepiej, aby wartość ta nie przekraczała 8.

Czułość odbiornika (RxGain) oraz zasilanie:

  • set radio.rxgain on/off oraz set radio.fem.rxgain on/off: Opcje te kontrolują wewnętrzne i zewnętrzne wzmocnienie odbioru sygnału radiowego (LNA / Front-End Module).
    Kiedy warto włączyć: Gdy repeater jest zamontowany w miejscu o bardzo niskim poziomie szumów (czyste tło radiowe), aby wyłapywać skrajnie odległe i słabe sygnały.
    Kiedy wyłączyć (off): Gdy urządzenie pracuje w pobliżu masztów GSM/BTS lub innych silnych źródeł zakłóceń – dodatkowe wzmocnienie może przesterować odbiornik.
    Uwaga: Włączenie opcji rxgain zwiększa pobór prądu stacji, co należy uwzględnić przy zasilaniu bateryjnym!
  • powersaving on/off: Bardzo ważna funkcja dla posiadaczy repeaterów z zasilaniem bateryjnym/solarnym. Włączenie tej opcji usypia radio w momentach braku aktywności, co pozwala zaoszczędzić olbrzymie ilości energii.
    Dla głębokiej optymalizacji zużycia baterii niezwykle przydatny może okazać się również specjalny, zoptymalizowany firmware od IoTThinks EasySkyMesh: https://github.com/IoTThinks/EasySkyMesh/releases.

Ustawienia opóźnień nadawania (txdelay):

Zróżnicowanie czasów opóźnienia nadawania (parametr txdelay) chroni sieć przed kolizjami radiowymi na tym samym obszarze (tzw. zjawisko ukrytego węzła). Im wyżej położony i lepiej słyszalny jest dany przemiennik, tym dłużej powinien odczekać przed retransmisją, pozwalając na nadanie stacjom położonym niżej.

Profil wysokości / Sąsiedzi Opis zastosowania Zalecane Opóźnienia (CLI)
WYSOKI / MASZT

set direct.txdelay 1.0
set rxdelay 3

Sąsiedzi: 5 do 15		 Klasyczny montaż anteny na dachu domu jednorodzinnego lub niskiego bloku.

set rxdelay 3

11. Podsumowanie: Kompletna lista domyślnych komend CLI

Poniżej zebraliśmy wszystkie rekomendowane i standardowe komendy konfiguracji repeatera na Zachodzie w jednym miejscu. Możesz skopiować całą listę lub poszczególne parametry do wdrożenia w konsoli. Pamiętaj, aby po wszystkim zrestartować urządzenie: 

# 1. Tożsamość i kontakt (wprowadź swój własny nick)
set owner.info "TWÓJ_NICK_LUB_ZNAK"

# 2. Czasy rozgłoszeń (adverty)
set advert.interval 240
set flood.advert.interval 23
set flood.max.advert 8

# 3. Synchronizacja zegara
clock sync

# 4. Hasło gościa (puste)
set guest.password ""

# 5. Tryb przekazywania (Repeat Mode)
set repeat.mode on

# 6. Optymalizacja routingu i parametrów LoRa oraz duty cycle
set dutycycle 10
set path.hash.mode 2
set multi.acks 1
set agc.reset.interval 500
set flood.max 64
set flood.max.unscoped 8

# 7. Konfiguracja zakresów regionalnych (Scopes)
region put pl
region put zachod
region put pl-ds
region put pl-lb
region put wlkp
region put wroc
region allowf pl
region allowf zachod
region allowf pl-ds
region allowf pl-lb
region allowf wlkp
region allowf wroc
set default.region zachod
region save

# 8. Opcjonalne oszczędzanie energii (odkomentuj jeśli zasilasz bateryjnie)
# powersaving on

# Na koniec zrestartuj urządzenie (komenda "write" nie występuje w urządzeniach):
reboot
