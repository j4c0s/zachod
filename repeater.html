Instrukcja dla Repeaterów

Parametry techniczne, zalecane limity przeskoku (flood), opcje optymalizacyjne oraz profile opóźnień (txdelay) dla stacji bazowych na Zachodzie.

1. Kroki kontrolne przed montażem

Przed umieszczeniem przemiennika wysoko na dachu lub maszcie, upewnij się, że stacja działa bez zarzutu pod kątem podstawowej diagnostyki:

  • Zasilanie i pobór prądu: Sprawdź stabilność obciążenia akumulatorów, przetwornic lub zasilacza buforowego.
  • Poziom szumów (Noise Floor): Zmierz wskaźnik RSSI tła w CLI. Ciągły poziom szumów powyżej -110 dBm oznacza lokalne zakłócenia utrudniające odbiór. Jeśli w pobliżu znajdują się maszty telefonii komórkowej BTS, pomocne może okazać się zastosowanie dedykowanego filtra szczelinowego (cavity filter) na pasmo 868 MHz.

2. Nazewnictwo Repeaterów

Dobra praktyka: Ze względu na brak sztywnego krajowego standardu, najważniejsza jest jasna lokalizacja pozwalająca określić położenie geograficzne węzła. W nazwie nie musi znajdować się typ montażu ani oznaczenie "RPT". Wystarczy prosta i czytelna nazwa, np. Wroclaw_Krzyki lub Luban_Srednia, co pozwala innym użytkownikom łatwo zidentyfikować pozycję na mapie i w logach.

3. Rozgłoszenia (Adverty)

Aby zapobiec wysycaniu pasma zbyt częstymi, powtarzającymi się pakietami statusowymi, konfigurujemy rzadkie interwały dla obu typów advertów (nie nazywamy ich "reklamami"):

  • Advert Lokalny (Zero-hop): Status wysyłany wyłącznie do bezpośrednich sąsiadów. Standardowo ustawiamy go na 240 minut (czyli równe 4 godziny).
  • Advert Globalny (Flood): Status rozsyłany kaskadowo (flood) po całej sieci. Dla odciążenia pasma konfigurujemy go na 23 godziny (oraz 0 hopów bezpośrednich).
# Status lokalny (zero-hop) wysyłany co 240 minut (4 godziny)
set advert.interval 240

# Status globalny (flood) przekazywany po sieci co 23 godziny
set flood.advert.interval 23

4. Limity Przeskoku (Hopy)

W celu zredukowania ogólnego szumu radiowego w eterze, wprowadzamy rygorystyczne limity na ruch bez zdefiniowanego zakresu regionalnego (unscoped), zachowując jednocześnie pełen szeroki zasięg dla pakietów ze scope regionalnym: 

# Maksymalny limit hopów dla pakietów ze scope regionalnym (np. #zachod)
set flood.max 64

# Ograniczenie przesyłu dla ruchu ogólnego bez scope (unscoped)
set flood.max.unscoped 8
💡 Dostosowanie parametru unscoped: Wartość flood.max.unscoped 8 jest domyślnym limitem. Możesz dostosować ten limit do swoich lokalnych potrzeb obserwując nadchodzące pakiety. Im niższa wartość unscoped, tym mniejsze niepotrzebne obciążenie dla Twojej lokalnej sieci. Zaleca się ustawienie parametru w bezpiecznym i optymalnym przedziale od 6 do 8.

5. Optymalizacja parametrów CLI oraz synchronizacja czasu

Poniższy zestaw komend optymalizuje routing, diagnostykę i stabilność pracy odbiornika, a także rozwiązuje problem z nieprawidłowym czasem systemowym:

  • set path.hash.mode 2: Włącza zaawansowany i bardziej unikalny tryb generowania skrótów ścieżek (path-hash), co znacząco zmniejsza ryzyko kolizji identyfikatorów tras.
  • set multi.acks 1: Aktywuje wielokrotne potwierdzenia odbioru pakietów, stabilizując przesył w trudnych warunkach propagacyjnych.
  • set agc.reset.interval 500: Zapobiega "zatykaniu" się czułego układu odbiorczego SX1262 przez silne pobliskie sygnały radiowe, wymuszając okresowe resetowanie automatycznej regulacji wzmocnienia (AGC).
  • set guest.password "": Pozostawia puste hasło gościa, co pozwala Twoim sąsiadom na bezpieczne odczytanie podstawowych parametrów stacji bez możliwości wprowadzania zmian.
  • set owner.info "..." : Ustawia czytelne dane kontaktowe właściciela stacji (np. Twój znak krótkofalarski, nick lub adres e-mail), ułatwiając współpracę w regionie.
  • Synchronizacja czasu: Prawidłowy czas jest kluczowy dla dopasowania logów i pakietów telemetrycznych. Użyj komendy clock sync w terminalu. Jeżeli czas urządzenia wybiegł w przyszłość i nie da się go cofnąć, wykonaj pełen reset zegara komendą clkreboot, a dopiero po nim wydaj komendę clock sync.
# Optymalizacja routingu i mechanizmów sieciowych
set path.hash.mode 2
set multi.acks 1
set agc.reset.interval 500
set guest.password ""

# Identyfikacja właściciela węzła (podaj np. Nick / Znak / Kontakt)
set owner.info "Tworzyciel_Zachod"

# Synchronizacja zegara systemowego (w przypadku problemów użyj najpierw clkreboot)
clock sync

# Zapis zmian w pamięci trwałej i restart
write
reboot

6. Profile Opóźnień Nadawania (txdelay)

Dopasowanie czasów opóźnienia nadawania (txdelay) jest krytycznym elementem ochrony sieci przed wzajemnym zagłuszaniem stacji (kolizje radiowe). Profil opóźnienia dobieramy bezpośrednio na podstawie szacowanej liczby widzianych sąsiadów radiowych oraz wysokości zawieszenia anteny.

Im wyżej położona antena i im więcej stacji ma w swoim bezpośrednim zasięgu, tym dłużej powinna odczekać (wyższe opóźnienie), pozwalając na swobodne nadanie mniejszym stacjom lokalnym znajdującym się na dole.

Profil wysokości Szacowana liczba sąsiadów Zastosowanie Parametry CLI
WYSOKI / MASZT Powyżej 15 sąsiadów Wysokie wieże, maszty kratownicowe, dachy wieżowców (bardzo daleki zasięg) set txdelay 1.5
set direct.txdelay 1.0
set rxdelay 3
DACH / ROOFTOP Od 5 do 15 sąsiadów Antena na dachu domu jednorodzinnego lub średniej wysokości bloku set txdelay 0.8
set direct.txdelay 0.4
set rxdelay 3
DOM / NISKI Poniżej 5 sąsiadów Węzeł domowy, antena na parapecie lub nisko w terenie zabudowanym set txdelay 0.3
set direct.txdelay 0.1
set rxdelay 3
