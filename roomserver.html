RoomServer / Observer

Jak skonfigurować bezgłośny punkt nasłuchowy (Observera) na ESP32 i połączyć go ze światową oraz lokalną telemetrią przez Wi-Fi.

Zasada działania Observera

Observer (obserwator) to urządzenie oparte o ESP32 z radiem LoRa, które nie wysyła własnych pakietów w eter (by nie generować niepotrzebnego szumu), lecz stale nasłuchuje pasma i przekazuje odebrane ramki drogą internetową (Wi-Fi) bezpośrednio na serwery telemetryczne i mapy topograficzne sieci.

Krok 1: Wstępna konfiguracja i Wi-Fi

Wgraj oprogramowanie MeshCore na urządzenie przy użyciu flashera USB. Następnie podłącz się do konsoli szeregowej CLI i skonfiguruj połączenie z domową siecią bezprzewodową Wi-Fi:

# Wprowadź dane Twojej domowej sieci bezprzewodowej
set wifi.ssid "Nazwa_Twojej_Sieci"
set wifi.password "Haslo_Do_Wifi"
set wifi.enabled true

Krok 2: Konfiguracja połączenia MQTT i serwerów telemetrii

Aby Twoja stacja skutecznie przekazywała pakiety na mapy i do baz analitycznych, aktywujemy przesyłanie danych MQTT. W regionie zachodnim wysyłamy dane jednocześnie do dwóch niezależnych baz:

  • letsmesh.net: Główna, globalna mapa sieci LoRa MeshCore.
  • meshcorekrk.cma.pl: Krajowy polski serwer zbierający statystyki jakości linków, poziomu szumów oraz trasowania pakietów.

Wprowadź w CLI poniższe komendy:

# Włączenie podstawowego przesyłu danych na letsmesh.net
set mqtt.enabled true
set mqtt.server "letsmesh.net"
set mqtt.port 1883

# Aktywacja raportowania na polski serwer analityczny
set custom.telemetry.server "meshcorekrk.cma.pl"
set custom.telemetry.enabled true

# Zapisanie ustawień i restart urządzenia
write
reboot

Po restarcie urządzenie automatycznie połączy się z siecią Wi-Fi i zacznie przesyłać odebrane pakiety radiowe do systemów internetowych. Na mapach Twój węzeł pojawi się jako aktywny punkt nasłuchowy oznaczony jako Observer.
